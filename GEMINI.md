# プロジェクト概要: Kengo Hoi — Portfolio

このプロジェクトは、**Astro v5** を使用して構築された、個人的なポートフォリオサイトです。
シングルページアプリケーション（SPA）のようなスムーズな閲覧体験を重視した構成で、スキルセット、経歴、制作実績（Works）を1ページに集約して見せるデザインになっています。

## 1. プロジェクトの概要と技術スタック

*   **フレームワーク:** [Astro](https://astro.build/) (v5.9.2)
    *   静的サイト生成（SSG）をベースにしており、高速なパフォーマンスを実現しています。
*   **言語:** TypeScript (`.astro` ファイル内のスクリプトや設定ファイルで使用)
*   **スタイリング:**
    *   **CSS Variables:** テーマ（ライト/ダークモード）の切り替えや共通カラーの管理に使用。
    *   **Scoped CSS:** 各 `.astro` コンポーネント内に記述されたCSSは、自動的にそのコンポーネントだけに適用されるため、スタイルの競合を防いでいます。
*   **コンテンツ管理:**
    *   **ハードコード:** メインページ（`index.astro`）の「Skills」や「Works（モーダル表示）」の多くは、直接ファイル内に記述されています。
    *   **Content Collections:** `src/content/work` ディレクトリ内の Markdown ファイルを管理する仕組みも設定されており（`src/content.config.ts`）、個別の作品ページ（`/work/[slug]`）を生成する機能も保持しています。

## 2. ディレクトリ構成と役割

主要なディレクトリとファイルの役割は以下の通りです。

*   **`src/pages/`**: ページのルーティングを決定します。
    *   `index.astro`: **サイトのトップページ**です。Heroセクション、About、Skills、Works（モーダル実装）のすべてがここに集約されています。
    *   `work/[...slug].astro`: Markdown ファイルから個別の作品詳細ページを動的に生成するルートです。
*   **`src/layouts/`**: 共通のページレイアウトです。
    *   `BaseLayout.astro`: `<head>`タグ、ナビゲーション（`Nav`）、フッター（`Footer`）、背景エフェクトなど、全ページ共通の枠組みを提供します。
*   **`src/components/`**: 再利用可能な UI 部品です。
    *   `Nav.astro`: ページ上部のナビゲーションバー。ページ内リンク（`#about`, `#works`）を管理し、レスポンシブ対応（スマホではハンバーガーメニュー）しています。
    *   `Hero.astro`: 各セクションのタイトルを表示するコンポーネント。
    *   `ThemeToggle.astro`: ライトモードとダークモードを切り替えるスイッチ。
    *   `PortfolioPreview.astro`: （現在は `index.astro` で直接記述されているため、使われていない可能性がありますが）作品カードのコンポーネントと思われます。
*   **`src/styles/`**:
    *   `global.css`: サイト全体の配色（カラーパレット）、フォント、リセットCSS、ダークモード用の変数定義などが記述されています。
*   **`public/`**: 静的ファイル置き場。
    *   `assets/`: 背景画像やテクスチャ素材。
    *   `img/`: 作品のスクリーンショットや動画ファイル。

## 3. 主要な機能と実装のポイント

### A. シングルページ構成とモーダル表示 (`index.astro`)
トップページは長いスクロールページとなっており、ナビゲーションをクリックすると各セクションへスムーズにスクロールします。

*   **Worksセクション:**
    *   通常、ポートフォリオサイトでは「詳細ページへ遷移」することが多いですが、このサイトでは **`<dialog>` タグを用いたモーダルウィンドウ** で詳細を表示する設計になっています。
    *   これにより、ユーザーはページ遷移することなく、素早く複数の作品詳細を閲覧できます。
    *   モーダル内のコンテンツ（研究内容、システム構成、動画など）は `index.astro` 内に直接 HTML で記述されています。

### B. テーマ切り替え（ライト/ダークモード）
*   `src/styles/global.css` にて、`:root`（ライトモード）と `:root.theme-dark`（ダークモード）の2つの変数セットが定義されています。
*   `BaseLayout.astro` や `ThemeToggle.astro` のスクリプトにより、ユーザーの選択やシステム設定（`prefers-color-scheme`）に応じて `<html>` タグにクラスを付与し、見た目を切り替えています。

### C. スキルセットの表示
*   `index.astro` 内の `const skills = [...]` 配列に、Blender, Python, C, Unity などのスキルと習熟度（レベル）、説明文がデータとして定義されています。
*   これを `.map()` 関数でループ処理し、星（★）の数でレベルを可視化したカードとして表示しています。

### D. コンテンツコレクション (`src/content/`)
*   `src/content.config.ts` で `work` コレクションが定義されています。
*   `src/content/work/` 内の `.md` ファイルを読み込みますが、現在のトップページ（`index.astro`）の「Works」セクションはハードコードされた内容が優先されているようです。Markdown ファイルは `/work/[slug]` という別URLでアクセス可能なページとして生成されていますが、メインの動線はモーダル（トップページ内）になっています。

## 4. デザインと演出
*   **背景:** `BaseLayout.astro` にて、ノイズテクスチャ（`noise.png`）とグラデーション画像を重ね合わせることで、質感のあるリッチな背景を作り出しています。また、スクロールに合わせてふわふわ動く円のアニメーション（`bg-circles`）も実装されています。
*   **レスポンシブ:** メディアクエリ（`@media (min-width: 50em)` など）を多用し、スマホとPCでレイアウト（グリッドの列数やナビゲーションの形状）が最適化されるよう設計されています。
