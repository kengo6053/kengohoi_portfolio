# プロジェクト概要: Kengo Hoi — Portfolio

このプロジェクトは、**Astro v5** を使用して構築された、個人のポートフォリオサイトです。
シングルページアプリケーション（SPA）のようなスムーズな閲覧体験を提供し、スキルセット、経歴、制作実績（Works）を1ページに集約して表示しています。

## 1. プロジェクトの概要と技術スタック

*   **フレームワーク:** [Astro](https://astro.build/) (v5.9.2)
    *   静的サイト生成（SSG）による高速なパフォーマンスを実現。
*   **言語:** TypeScript
*   **スタイリング:**
    *   **CSS Variables:** テーマ（ライト/ダークモード）の切り替えや共通カラーの管理。
    *   **Scoped CSS:** コンポーネント単位のスタイル管理による競合防止。
*   **アニメーション & インタラクション:**
    *   **Scroll Reveal:** `IntersectionObserver` を使用し、スクロールに合わせて要素がふわっと表示される演出を実装。
    *   **Glassmorphism:** ヘッダーに透過背景とブラー効果（`backdrop-filter`）を採用し、モダンな質感を演出。
*   **コンテンツ管理:**
    *   **Content Collections:** `src/content/work` 内の Markdown ファイルを使用して制作実績を管理。
    *   **Skills:** `src/components/SkillsList.astro` 内の配列データとして定義。

## 2. ディレクトリ構成と役割

主要なディレクトリとファイルの役割は以下の通りです。

*   **`src/pages/`**:
    *   `index.astro`: **サイトのトップページ**。動的な背景演出、Scroll Reveal の制御スクリプト、広めにとられたヒーローセクションを統合。
*   **`src/components/`**:
    *   `Nav.astro`: **透過型固定ヘッダー**。`sticky` 配置され、背後の動的な背景が透けて見えるグラスモーフィズムデザイン。
    *   `WorksSection.astro`: 制作実績セクション。カテゴリ別に分類して表示。
    *   `WorkCard.astro`: 作品カード。スクロール連動アニメーションを適用。
    *   `WorkModal.astro`: `<dialog>` による詳細表示。
    *   `SkillsList.astro`: カテゴリ別スキル表示。表示タイミングをずらす「スタッガー効果」を実装。
*   **`src/styles/`**:
    *   `global.css`: 全体のスタイル定義。Scroll Reveal 用のアニメーション定義（`.scroll-reveal`, `.in-view`）を含む。

## 3. 主要な機能と実装のポイント

### A. スクロール連動アニメーション (Scroll Reveal)
サイト全体の質感を高めるため、以下の演出を導入しています。
*   **フェードイン & アップ:** 要素が画面内に入ると、不透明度と位置が滑らかに変化します。
*   **スタッガー（順次表示）:** スキルカードなどにおいて、左から順に時間差で表示されるよう制御し、リズム感を与えています。

### B. グラスモーフィズム・ナビゲーション
ヘッダーを透過させることで、常に背後の動的な背景演出（水玉）が楽しめるようになっています。
*   **高い透過度:** 背景を `hsla(..., 0.4)` まで透過させつつ、`backdrop-filter: blur(12px)` で可視性を確保。
*   **自然な境界:** はっきりした境界線を排除し、薄いシャドウと余白調整によりコンテンツと一体化させています。

### C. 動的な背景演出
*   `index.astro` にて、JavaScript を使用してランダムな円（`.bg-circle`）を生成。
*   スクロール量に応じたパララックス（視差効果）アニメーションにより、奥行きのある視覚体験を提供。

### D. データ駆動型の Works セクション
制作実績はすべて Markdown で管理され、カテゴリ（Programming / Visual）への自動振り分けやモーダル表示に対応しています。

## 4. デザインと演出
*   **余白設計:** ヒーローセクションに十分な上下余白（`padding-block: 12rem` 等）を確保し、タイトルのインパクトを強めています。
*   **アクセシビリティ:** キーボード操作への対応や、`prefers-reduced-motion` への配慮、モーダル内のフォーカス管理を実装。
